1. 系统概览

FishAutoPro 是一个面向闲鱼多账号运营的自动化管理平台，覆盖账号接入、商品/订单/会话管理、自动回复、自动发货、AI知识库、通知、许可证、运维管理等完整链路。

平台支持：支持 Windows、macOS、Linux 和 Docker 平台部署。

角色：超级管理员 角色：普通用户 强权限模块已做超级管理员限制

模块类别 主要页面 说明
业务运营 仪表盘、账号、商品、订单、对话 日常运营核心页面，支持按账号筛选、实时状态同步。
自动化 自动回复、AI知识库、自动发货、发货流程 支持规则化与 AI 化自动处理能力。
系统管理 通知、设置、许可证、免责声明、返利 配置平台行为与商业化能力。
运维与安全 用户管理、邮件服务配置、系统日志、数据备份 仅超级管理员可见与可操作。

2. 登录注册与免责声明

2.1 登录方式

  • 账号 + 密码 + 图形验证码。
  • 邮箱 + 密码 + 图形验证码。
  • 邮箱 + 邮箱验证码（免图形验证码）。

2.2 注册流程

  • 注册时需填写账号、邮箱、密码、图形验证码、邮箱验证码。
  • 邮箱验证码发送前会校验邮箱格式、注册用途与图形验证码。

2.3 免责声明机制

  • 用户首次登录后，如未同意免责声明，将强制跳转免责声明确认页。
  • 用户勾选并提交后才可进入系统首页。
  • 系统同时提供“免责声明”独立菜单页用于长期查看。

2.4 超级管理员来源

  • 超级管理员账号由 config.json 的 auth.superAdmin.username/password 同步生成。
  • 业务页面不提供创建或修改超级管理员入口。

3. 仪表盘

  • 展示账号在线数、消息会话数。
  • 订单与金额统计：全部/近7天切换，且支持全部账号或指定账号统计。
  • 提供月度、今日维度的订单金额统计（已成交、未成交、已支付等）。
  • 提供“待发货数量”指标。

4. 账号管理

  • 支持 Cookie 新增/编辑账号，支持启停、重启、刷新资料。
  • 支持账号备注、自动好评开关与账号级自动好评文案配置。
  • 删除账号时提供危险确认弹窗，并明确提示将删除的关联数据范围。
  • 后端删除逻辑会清理订单、会话消息、规则、日志、通知绑定等账号关联数据。

5. 商品管理

  • 支持按账号、状态筛选商品。
  • 支持分页加载与刷新。
  • 商品数据可用于自动回复、自动发货规则中的“按商品生效”配置。

6. 订单管理

  • 支持按账号、状态筛选订单。
  • 支持手动按账号 + 订单号拉取订单详情。
  • 支持单笔刷新状态、发货、免发货、删除。
  • 支持多选与批量删除订单。

7. 对话消息

  • 会话列表支持按账号、商品筛选，支持批量删除会话。
  • 消息明细支持加载更早消息、消息多选删除。
  • 支持发送文字与图片；图片支持本地上传。
  • 支持直接粘贴剪贴板图片到输入框区域，先预览再发送。
  • 发送图片会携带真实宽高，避免客户端按默认比例变形。
  • 会话列表支持展示首次商品快照（标题/图片），降低商品被删后的识别成本。

8. 自动回复

  • 规则支持按账号生效（全局或指定账号）。
  • 支持按商品作用域与“排除商品”配置。
  • 匹配方式：精确匹配、包含关键词、正则匹配、AI 回复。
  • 支持“排除匹配”兜底规则。
  • 回复类型支持文本和图片；图片回复支持上传并保存宽高。
  • 图片回复支持“前置文字”，即先发文字再发图片。
  • AI 回复可绑定AI知识库，未填提示词可回退全局提示词。

9. AI知识库

  • 支持新建知识库并上传文本类文件（如 txt/md/json/csv/srt/vtt）。
  • 支持向已有知识库追加文档、删除文档、删除知识库。
  • 支持知识块检索测试，用于验证召回效果。

10. 自动发货

  • 规则支持按账号、商品、商品规格匹配。
  • 发货类型支持固定文本、库存发货、API 取货。
  • 触发时机支持付款后/确认后。
  • 可绑定“发货流程”作为执行链路。
  • 库存发货支持库存导入、查看、清理与已用库存管理。

11. 发货流程

  • 可视化流程编辑（思维导图方式），支持新增子节点、兄弟节点、条件分支。
  • 支持节点参数配置（延迟、条件、消息等）。
  • 支持保存/设为默认流程。
  • 支持流程导入/导出 JSON，导出 PNG。

12. 消息通知

  • 通知渠道管理：企业微信、钉钉、飞书/Lark、Bark、Email SMTP、Webhook、Telegram。
  • 渠道支持启停、编辑、删除、发送测试通知。
  • 支持“账号-通知渠道”绑定，可按账号独立管理消息通知策略。

13. 系统设置

  • 主题切换。
  • 免拼单自动免拼发货开关与延迟配置（卡片消息触发后自动调用免拼发货接口）。
  • 自动好评全局文案与接口配置（账号级文案为空时回退全局）。
  • AI 服务配置（地址、模型、密钥、系统提示词）与连接测试。

14. 超级管理员专属模块

以下模块前后端均限制超级管理员访问。

  • 用户管理：查看用户列表，删除普通用户（不可删除超级管理员）。
  • 邮件服务配置：配置 SMTP，测试连通性，清空 SMTP 密码。
  • 系统日志：查看实时日志/历史日志，支持日期、文件、级别筛选。
  • 数据备份：数据库导出、导入恢复、手动重启服务。

15. 许可证管理

  • 展示授权状态、到期信息、机器码。
  • 支持导入 JSON 许可证。
  • 提供机器码复制能力，便于线下发证。

16.1 授权价格

  • 1个月 ¥7
  • 半年 ¥35
  • 一年 ¥50
  • 永久 ¥100

16.2 返利提成规则

推广一个人购买一单，就可以得到成交订单金额的 30% 提成。

例如：推广一个人买了 100 元永久授权，可得到 30 元提成。

  • 展示邀请码与返利说明文案。
  • 支持展示返利联系二维码，便于业务转化。

17. 移动端适配

  • 提供移动端独立布局（抽屉菜单）。
  • 移动端右上角展示当前用户头像、用户名、角色，并支持退出登录。
  • 核心菜单与权限显示规则与 PC 端保持一致。

18. 多账号独立与权限隔离

18.1 账号数据归属

  • 账号、自动回复、自动发货、知识库、通知渠道、发货流程等核心数据带 owner_user_id 归属。
  • 普通用户仅可访问自己归属的账号与相关资源；超级管理员可全局访问。

18.2 按账号可见范围联动

  • 订单、会话、消息等通过“可访问账号列表”进行访问控制，确保用户仅看到自己账号的业务数据。
  • WebSocket 推送同样按可访问账号范围过滤。

18.3 高权限接口隔离

  • 数据备份、用户管理、系统日志、邮件服务配置接口均有超级管理员鉴权。
  • 前端路由与菜单也同步做了超级管理员可见性控制，避免误触达。

19. 账号可用性告警逻辑

系统内置账号可用性监控，采用“两条告警线”覆盖不同故障阶段，并可通过通知渠道推送。

19.1 重连失败告警（reconnect_stuck）

  • 触发场景：账号重连过程中持续失败，始终无法恢复在线。
  • 触发阈值：连续失败达到 3 次触发告警。
  • 冷却机制：同一账号 3 小时内最多重复告警 1 次。
  • 恢复机制：账号重新成功上线后，失败计数自动清零。

19.2 Token 刷新失败告警（token_refresh）

  • 触发场景：运行中 Token 刷新持续失败（主要针对 cookie 失效、风控类错误）。
  • 触发阈值：10 分钟窗口内失败次数达到 3 次，或连续失败达到 3 次。
  • 冷却机制：同一账号 3 小时内最多重复告警 1 次。
  • 保护动作：达到告警阈值后，系统会自动停用该账号并断开连接，避免持续异常重试。
